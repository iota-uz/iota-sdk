# Projects

Projects helps teams organize planned work, track progress, and keep project stages visible. It is useful for internal initiatives, client work, and operational programs.

## Who uses it

- Project managers
- Team leads
- Operations teams
- Client delivery teams

## What it manages

- Projects
- Project stages
- Project status and progress information

## Why it matters

Projects give teams a shared view of what is active, what stage each effort is in, and where attention is needed. This reduces status meetings based only on memory and helps managers compare work across teams.

## Common tasks

- Create a project.
- Move a project through stages.
- Review active projects by status.
- Update project details before a planning meeting.
- Maintain the list of stages used by the organization.

## Best practices

- Keep stages simple and meaningful.
- Update project status as part of the working routine.
- Archive or close completed work when it is no longer active.
- Use consistent project names for easy search.
