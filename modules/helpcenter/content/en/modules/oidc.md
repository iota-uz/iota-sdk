# OIDC Single Sign-On

OIDC connects the ERP to an external identity provider so users can sign in with their organization account. It supports centralized identity and a smoother login experience.

## Who uses it

- IT administrators
- Security teams
- Platform administrators
- Organizations with centralized identity management

## What it manages

- Identity clients
- Sign-in redirects
- Authentication requests
- User identity claims
- Integration with external identity providers

## Why it matters

Single sign-on reduces password sprawl and helps organizations manage access from one identity system. It is especially useful when employees already use a central provider for email, workplace tools, or directory access.

## Common tasks

- Configure an identity provider.
- Test the sign-in flow.
- Review which users can access the ERP.
- Coordinate account lifecycle with IT policies.

## Best practices

- Use SSO for employee access when possible.
- Keep fallback administrator access protected.
- Review identity provider settings before rollout.
- Coordinate role assignment in Core Administration after sign-in is working.
