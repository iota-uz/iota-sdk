# Seeding (Legacy)

Legacy seed/e2e commands were removed in the dbctl cutover.

Use `/getting-started/dbctl` for all seed and e2e database workflows.

```bash
# Plan
command dbctl plan seed.main --yes --approve-ticket local-dev

# Apply
command dbctl apply seed.main --yes --approve-ticket local-dev
command dbctl apply seed.superadmin --yes --approve-ticket local-dev
command dbctl apply db.e2e.reset --yes --approve-ticket local-dev
```
